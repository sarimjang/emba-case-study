# EMBA Case Analysis Template

Use this template when the user wants a full case writeup, class-prep brief, or structured decision memo.

## 一、導言與決策切入點

- 核心主角是誰
- 他或她此刻必須決定什麼
- 決策點前的 3-6 個關鍵事件里程碑
- 為何不能再拖
- 延遲決策的代價是什麼

建議收尾句：
`本個案真正的張力，不只是事件本身，而是主角必須在資訊不完全、時間有限、組織壓力升高的情況下做出關鍵選擇。`

## 二、公司與產業總體檢

### 企業背景

- 成立脈絡與發展階段
- 核心產品、客群、收入模式
- 競爭優勢與已知弱點

### 產業背景

- 市場規模與成長性
- 競爭格局
- 法規、技術、通路或地緣限制

### 多維度拆解

- 市場與通路：需求變化、通路依賴、品牌或行銷困境
- 成本與供應鏈：成本結構、原料或產能瓶頸、供應中斷風險
- 技術與組織：技術缺口、文化衝突、跨部門協作障礙

### 深層矛盾

- 真正的大問題是什麼

### Trade-off 定義

- 選項 A 與選項 B 的矛盾是什麼
- 組織真正不能同時擁有的是什麼

## 四、數據驗證與實例支持

### 個案內部證據

- 財報或管理報表
- 市調圖表
- 組織圖
- 訪談或備忘錄

### 計算假設（寫在對應計算正上方，不要事後才補在附錄）

- 本節如果有任何推導出來的數字（非個案原文直接給的），先寫出用了什麼假設
- 例如：「以食物成本 33%、飲料成本 20%、食物:飲料=70:30 為假設」
- 產出對應的可重跑腳本與計算紀錄 md（`calculations/<calc-id>.py` + `.md`），並在附錄「參考來源」列出路徑

### 內部驗證

- 重要數字是否可重算
- 圖表結論是否跟原始數字一致
- 是否可推導毛利、單位成本、產能利用率、現金壓力

### 外部交叉比對

- 法規文件
- 同業年報或法說資料
- 官方產業報告
- 可信研究機構資料

### 決策 Checkpoint

所有選項與兩層 trade-off 攤開後，先停下來問使用者：

- 你會選哪一個選項？
- 或者，你認為哪些決策標準／權重最重要？

只有在使用者明確要求「你直接決定」時，才由模型自行拍板並說明理由。

### 決策結論

一定要根據上一步的 checkpoint 結果，寫出一個明確決策，不能留成中立的討論題組。決策對錯不是重點，但必須做出決策；若使用者仍想保留課堂討論空間，可以在決策之後**額外**附上開放式討論題，而不是用討論題取代決策。
