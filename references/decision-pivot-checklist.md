# Decision Pivot Checklist

Use this before writing the main analysis. If this section is weak, the whole case analysis becomes a generic summary.

## A. Protagonist

- 誰有正式決策權
- 誰承擔結果責任
- 誰可能影響決策但不是最後拍板者

## B. Decision Point

Express the decision as one sentence:

`主角必須在 {期限/事件} 前，決定是否/如何 {核心行動}，以回應 {關鍵壓力或危機}。`

Examples:
- CEO 必須在下一輪董事會前，決定是否出售虧損事業部，以避免現金流危機擴大。
- 區域總經理必須在主要客戶續約前，決定是否降價換取市占，以回應競爭者的價格戰。

## C. Timeline

Build a short milestone chain:
- 起點：問題何時開始形成
- 轉折：哪個事件使問題升級
- 臨界點：為何現在變成非做不可

Minimum useful output:
1. `T-3` 或更早的背景事件
2. `T-2` 的惡化訊號
3. `T-1` 的觸發事件
4. `T0` 的決策當下

## D. Urgency

Ask:
- 如果延後 1 週 / 1 季 / 1 年，後果是否不同
- 損失是財務性的、聲譽性的、法規性的，還是組織性的
- 主角是在回應危機，還是在搶時間窗口

## E. Output Test

This section is strong enough only if:
- you can state the decision in one sentence
- you can explain why now in two sentences
- you can name the cost of delay without vague wording
